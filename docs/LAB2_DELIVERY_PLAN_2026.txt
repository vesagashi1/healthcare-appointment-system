# Advanced Healthcare Management System
## Lab Course 2 Delivery Plan (Afati: Shkurt 2026)

## 1) Gjendja aktuale kundrejt kërkesave
- Frontend framework: `React` (OK)
- Backend framework: `Express.js` (OK)
- SQL database: `PostgreSQL` (OK)
- NoSQL database: `MUNGON` (duhet shtuar për notën e plotë në kërkesat obligative)
- Git workflow: ekziston repo, duhet rritur disiplina e branch/PR/commit convention
- Project management tool: duhet të përdoret dhe dokumentohet (Trello/Jira/ClickUp)

## 2) Veçori shtesë të rekomanduara (maksimizim i pikëve)
Zgjidhni të paktën 3 nga lista, me prioritet:
1. Authentication + Authorization me `access + refresh tokens` (zëvendëson modelin aktual single JWT)
2. Real-time Notifications me WebSockets (njoftime për termine: created/approved/cancelled)
3. Dynamic Report Generation + eksport (raporte me filtre date/rrole/ward)
4. Advanced Search (minimum 5 lista me filtra + sort + search)
5. Data Export/Import (e keni bazën; ta ngrisim në nivel production me validime/audit)

## 3) Ndarja e punës për ekip 2-personësh

### Dev 1 (Backend + Data + Security)
- Implemento refresh token flow:
  - tabela `refresh_tokens` (hashed token, user_id, expires_at, revoked_at)
  - endpoint-e: `/api/auth/login`, `/api/auth/refresh`, `/api/auth/logout`
  - rotating refresh tokens + revocation
- Integrim NoSQL (MongoDB rekomandohet):
  - koleksion `notifications`
  - koleksion `activity_events` ose `audit_events_extended`
  - sinkronizim eventesh nga backend SQL -> NoSQL
- WebSocket gateway:
  - `socket.io` server
  - rooms sipas `userId`/`role`
  - emit evente për appointment lifecycle
- Hardenim API:
  - input validation (`zod`/`joi`)
  - rate limiting për auth endpoints
  - uniform error handler + request id

### Dev 2 (Frontend + UX + Reporting)
- UI për auth të ri:
  - refresh handling në `api` layer
  - logout i sigurt në token expiry/revoke
- Real-time notifications UI:
  - notification bell + unread count
  - live toast/center panel për evente termine
- Advanced search:
  - të paktën 5 lista: users, patients, doctors, appointments, records
  - filtra + sort + debounce text search
- Dynamic reports:
  - faqe raporte me filtre (date range, role, ward, status)
  - eksport në CSV/XLSX/JSON

## 4) Definition of Done (DoD)
- Kërkesat obligative të plotësuara (përfshi NoSQL)
- 3 veçori shtesë të implementuara dhe demonstruara live
- Seed + setup i riprodhueshëm nga README
- Testim minimal:
  - backend integration tests për auth + appointments + access control
  - frontend smoke tests për route guards + kritike flows
- Dokumentim dorëzimi:
  - arkitekturë, endpoints kryesore, skema SQL + NoSQL, screenshot-e, video demo e shkurtër

## 5) Sprint plan (2-3 javë)
- Sprint 1:
  - Refresh token architecture
  - MongoDB integration
  - WebSocket skeleton
- Sprint 2:
  - Notifications end-to-end
  - Advanced search (5 lists)
  - Reports + export polish
- Sprint 3:
  - Stabilizim, testim, bugfix, dokumentim final

## 6) Praktika profesionale (duhet ndjekur)
- Branch strategy:
  - `main` protected
  - feature branches `feature/<scope>`
- PR template:
  - what/why, screenshots, test evidence, rollback note
- Commit convention:
  - `feat:`, `fix:`, `refactor:`, `docs:`, `test:`
- Task board:
  - Backlog -> In Progress -> Review -> Done
  - çdo task me owner + due date + acceptance criteria
